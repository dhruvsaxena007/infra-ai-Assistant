# Assistant Brain Stabilization — Routing Audit Report

Generated by `scripts/run_routing_audit.py`. Re-run after routing changes.

## Root causes fixed

| Issue | Cause | Fix |
|-------|--------|-----|
| RAG on greetings / search / recommendations | `_should_prefer_rag_over_search()` hijacked `unknown` + `machine_recommendation` when session had docs | **Removed** broad RAG prefer path |
| RAG blocked multi-purpose advisory | `multi_purpose_advisory` skipped when docs loaded | Removed `not rag_has_documents` guard |
| Doc upload questions → recommendation | Advisory layer ran before RAG for content questions | `_try_session_document_rag()` **before** contextual advisory |
| Catalog dump on project recommendation | `list_all: True` on recommendation search | `list_all: False` + normal limit |
| "compaction" lost context | Pending project/purpose not resumed early | `_try_continue_pending_clarification()` at router entry |
| "I need a machine" → unknown | `unknown` is NON_SEARCH → support path before clarify | Early `is_broad_vague_query()` → clarification search |
| Weak purpose mapping | Scattered `_PURPOSE_CATEGORIES` | Central **`PURPOSE_TO_MACHINE`** + **`PURPOSE_ALIASES`** |

## RAG activation rules (after stabilization)

RAG runs **only** when:

1. `intent == document_question`, OR
2. User **explicitly references** uploaded document (`_references_uploaded_document`), OR
3. Session has docs **and** question is **document content** (`_is_document_content_question`) — e.g. *"What machines are needed for highway project?"* after upload

RAG never runs for: greeting, machine search, recommendation, support, refund, payment, advisory.
